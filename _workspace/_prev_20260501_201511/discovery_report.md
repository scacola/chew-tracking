## Executive Summary

**한 줄 결론**: AirPods IMU 기반 저작·식사속도 트래킹은 *기술과 객관 페인이 충분히 견고*하지만, *주관적 인식의 약함과 Apple OS 흡수 위험*이라는 두 그림자를 가진 **조건부 GO** 영역이다. 진입한다면 "씹기 트래커"가 아니라 *결과 지향(위 건강·체중) 친절 코치*로, 그리고 *오늘부터 8주짜리 검증 사이클*로 가야 한다.

**핵심 권고**: **조건부 GO** — 옵션 B "Slow Bites" (혼밥 시청 직장인용 위 건강 코치)를 1순위로, 옵션 D "ChewSync Pro" (B2B SaaS)를 *피벗 보험*으로 병행 설계. 옵션 A·C·E는 V2 모듈로 후순위.

**의사결정의 3대 근거**:

1. **기술과 시장의 *희귀한 동시 신호***: AirPods IMU 학술 baseline은 자유생활 F1 0.71–0.80 [기술-2.1]이고, 빠른 식사의 객관 페인은 비만 OR 2.15·대사증후군 OR 1.54·한국인 미란성 위염 +71% [시장-1.1·1.2]로 누적 메타분석으로 견고하다. 두 축이 *동시에* 강한 사례는 흔하지 않다.
2. **White Space가 정확히 비어있음**: 글로벌·한국 모두 "이어폰 IMU + 백그라운드 자동 + 친절한 톤 + 결과 프레이밍" 조합은 *완전히 비어있고* [경쟁-WhiteSpace], 직접 경쟁 13종은 모두 *약하거나 단종됨* [경쟁-2]. 죽은 거인들의 공통 실패 원인(정확도 과장·폼팩터 마찰·콘텐츠 부재)은 모두 *학습 가능*한 함정이다.
3. **Apple 흡수 위험은 회피 가능, 단 콘텐츠·페르소나 자산을 처음부터 만들 때만**: 2026.03 Apple jaw health 특허 [기술-1.2]는 핵심 위협이지만, Apple은 *raw 데이터 OS 기능*을 줄 뿐 *코칭·페르소나·한국 결과 프레이밍 콘텐츠*는 만들지 않는다. Eat Right Now가 임상 콘텐츠로 살아남은 모델 [경쟁-3.4] 차용 가능.

**즉시 다음 행동 1순위**:
> **이번 주 안에** [시장-페르소나#1·#2] 한지원·박소연형 사용자 *각 5명* 디스커버리 인터뷰 + AirPods Pro 2세대 IMU 프로토타입 데이터 수집(본인+지인 5명, N=20 식사) *동시 진행*. 8주 후 *실제 결제* 30%+ 게이트로 본격 GO/NO-GO 재평가. [제품-6.1·6.2]

## 2. 기술 타당성 요약

**결론**: `CMHeadphoneMotionManager`로 AirPods Pro/3세대/Max IMU 합법 접근 가능. iOS 한정이며, *25Hz·단일채널·백그라운드 회색지대*라는 3대 API 제약이 있으나 본 시나리오에 우연히 맞물린다. [기술-1.1, 6.1]

**핵심 정량**:
- 학술 baseline: 실험실 F1 **0.86–0.91**(IMChew 2024, EarBit lab) / 자유생활 F1 **0.71–0.80**(EarBit wild). 짧은 간식만은 0.45–0.53로 급락. [기술-2.1]
- 음향+IMU 융합 시 0.97+이지만, AirPods raw 마이크는 일반 앱에 미노출 → **AirPods의 이점은 IMU 위주에 묶인다**. [기술-2.3]
- 25Hz Nyquist 12.5Hz는 일반 저작 빈도(0.94–2Hz)에 충분. *bruxism은 불가, eating chewing은 OK*. [기술-5.4]

**정확도 KPI 권고**: 임상이 아닌 *행동 변화 코칭용 F1 0.75–0.85*. ±10–15% 오차는 "오늘 평소보다 30% 빨리 먹었어요" 메시지에 영향 없음. [기술-6.1]

**4개 결정적 한계와 mitigation**:
1. 식사 외 활동(말하기·걷기) 분리 → *식사 시작 사용자 트리거* 또는 audio session/시간컨텍스트 활용. [기술-5.1]
2. 사용자별 보정(LOSO 5–25pp 차이) → *온보딩 1–2식사 캘리브레이션*. [기술-5.2]
3. 좌·우 동시 IMU 불가 → *chewing count 단일 채널 한계 안에서 가치 제안*. [기술-5.3]
4. 백그라운드 보장 미흡 → *영상 시청 audio session 활성으로 우회 — 본 시나리오와 우연히 잘 맞음*. [기술-5.6]

**Apple 흡수 신호 (위협)**: 2022 bruxism 특허 + 2026.03 jaw health metric 특허("좌·우 헤드폰 각각 bite/chew 카운트"). Apple 자체가 이 영역을 헬스 기능으로 보고 있음. [기술-1.2, 5.7]

| 사례 | 실패 원인 | 우리에게 주는 함정 |
|---|---|---|
| **Vessyl** | 마케팅 데모 = 거짓말 (모든 음료 식별 약속 → 실제는 물만) | *측정 가능한 것만 약속하라*. "100% 모든 음식 검출" 같은 약속 절대 금지. |
| **HAPIfork** | 폼팩터 마찰 (한식·외식·여행 무용) | 전용 디바이스 = 카지노. AirPods는 이미 차고 있으니 안전 — 단 *AirPods 빼면 무용*이라는 동일 함정 주의. |

### 4.3 현재 성공 모범 — Eat Right Now (Dr. Jud Brewer) [경쟁-3.4]

- 28일 마음챙김 식사 코스 + 임상 입증(craving-eating −40%, 부정 폭식 −36%) + 의료 라이선스 채널.
- *센서 없이 콘텐츠만으로* 성공.
- **시사점**: 행동변화 = 트래킹 ≠ 측정. *코칭 메시지·일일 미션*이 결합되어야 28일 코스. **Eat Right Now에 없는 = 실시간 객관 데이터** = 우리의 차별화.

### 4.4 5대 페인 패턴 (1-2점 리뷰 추출) [경쟁-4]

1. *"정확도 거짓말"* — 마케팅 약속 vs 실제 격차 = 신뢰 붕괴 근원
2. *"구독 사기성 + 코치는 챗봇"* — Noom BBB 1,200+ 환불 분쟁
3. *"백그라운드·동기 유지 실패"* — 14일 후 70% 이탈. **AirPods 자동 검출이 정확히 이 페인을 죽임**
4. *"내 식문화에 안 맞음"* — 한국 사용자 다수. 우리의 *씹기 자체*는 식문화 중립
5. *"감시받는 느낌, 죄책감 유발"* — 식이장애 위험. *친절한 톤 필수*

### 4.5 White Space — 가장 매력적인 빈 공간 3개 [경쟁-5]

1. **백그라운드 자동 검출 + 혼밥 시청자 + 친절한 톤** (한국 1차 진입 정확)
2. **소화기/위식도역류/당뇨예방 의료 채널** (B2B 라이선스, 임상 근거 있음)
3. **Whoop/Oura의 회복 컨텍스트에 씹기 데이터 통합** (글로벌 헬스 옵티마이저)

## 5. 추천 제품 컨셉

### 5.1 1순위 — 옵션 B "Slow Bites" (Main) [제품-2 옵션B]

**한 문장**: *영상 보면서 혼자 점심·저녁을 빠르게 먹는 직장인*이 AirPods를 그대로 낀 채로, 자기도 모르게 식사 속도가 느려지고 위 건강·체중이 회복되는 *백그라운드 자동 코칭* 앱.

**타겟**: 한지원형 (위염, 1순위) + 박소연형 (다이어트 정체기, 2순위) 동시 커버.

**5축 비교 합계 23/25** (5개 옵션 중 최고). [제품-3]

## 6. 핵심 모순·결정 포인트

리서치 4개 사이의 *충돌*을 노출한다. 각 모순은 곧 의사결정 포인트다.

### 모순 1: 기술은 가능하다 ↔ Apple이 OS에 흡수할 수도 있다

- **증거 A** [기술-1.1, 2.1, 6.1]: `CMHeadphoneMotionManager`는 합법 접근, F1 0.71–0.80 자유생활 baseline 존재. 본 시나리오 백그라운드는 영상 audio session으로 우회 가능.
- **증거 B** [기술-1.2, 5.7]: 2026.03 Apple "Jaw Health Metric Using Headphones" 특허 — 좌·우 헤드폰 각각이 bite/chew 카운트와 비대칭 측정. Apple이 OS Health 앱에 흡수할 가능성.
- **결정 필요**: Apple이 같은 기능을 *raw 메트릭*으로 OS에 내장하면 우리 차별화는 무엇인가?
- **결정 옵션**:
  - (a) **콘텐츠·페르소나 자산 우선 구축** (Eat Right Now 모델) — 측정은 Apple이 줘도 *코칭은 못 줌*. → 옵션 B의 길.
  - (b) **임상 인증·B2B 채널 우선** (Recovery Record 모델) — Apple은 의료기관 워크플로우를 만들지 않음. → 옵션 A·D 길.
  - (c) **로컬 특화** (한국 식문화·언어) — Apple은 글로벌 OS 회사. → 옵션 E 길.
- **권고**: (a) + (c)의 V2 흡수 = 옵션 B 메인 + 한식 모듈. (b)는 18–24개월 후. [합성]

### 모순 2: 객관 페인은 강하다 ↔ 주관 인식은 결과 매개일 뿐

- **증거 A** [시장-1.1·1.2]: OR 1.5–2.15 메타분석 누적, 한국 미란성 위염 +71%. 의학적 "강"이 의심 여지 없음.
- **증거 B** [시장-2.1·2.3]: "씹기 횟수 트래킹" 자생 검색 거의 0. "알지만 안 함" 지배. 페인 인식이 *체중·소화기를 경유*해야만 들어옴.
- **결정 필요**: 사용자가 *스스로* 빠른 식사를 문제로 인식하지 않는데, 어떻게 매일 쓰게 만들 것인가?
- **결정 옵션**:
  - (a) **결과 KPI 직접 노출** ("위 건강 점수", "체중 정체 탈출 확률") — 행위 측정을 결과로 *번역해서* 보여줌. → 옵션 B의 핵심 UX 결정.
  - (b) **의료 트리거** (위염 진단 직후 의사 처방) — 인식이 약한 일반인 대신 *이미 진단받은 환자*만 타겟. → 옵션 A.
  - (c) **마음챙김 라이프스타일 프레이밍** — Calm 사용자처럼 *결과 없이도 가치를 사는* 세그먼트. → 옵션 C. 단 페인 4/10로 retention 위험.
- **권고**: (a) — 측정 단독 BM은 [시장-거짓수요#5] 거짓 수요 함정. 결과 프레이밍이 인접 카테고리(Noom/파스타) 검증된 BM 통로. [합성]

### 모순 3: 직접 경쟁사 13개 모두 약함/단종 ↔ 우리가 같은 함정에 빠질 위험

- **증거 A** [경쟁-1·2]: HAPIfork·Vessyl·Healbe·Bite Counter·20 Minute Eating·밀꼭 등 모두 좀비 또는 영세. *13종이 같은 페인을 풀려다 실패*했다는 사실은 시장 어려움 시그널.
- **증거 B** [경쟁-3·6]: 그러나 실패 원인이 *명확히 분류 가능*함 — 정확도 과장 / 폼팩터 마찰 / 콘텐츠 부재 / 카메라 의존 / 결과 부재. *각각 회피 가능한 함정*.
- **결정 필요**: 우리가 같은 함정을 피한다는 가설이 진짜인가, 자기 위안인가?
- **결정 옵션**:
  - (a) **MVP 출시 전 죽은 거인 패턴 체크리스트 통과 의무화** — 마케팅 약속 정직성 / 폼팩터 보유 디바이스 / 결과 KPI / 한식 호환 / 친절 톤. 5개 모두 충족 시 출시.
  - (b) **8주 베타 결제 게이트** — 4주차 결제 전환율 30%+ 미만이면 옵션 D 피벗 또는 가설 재구축. [제품-6.4]
  - (c) **포기** — 시장 진입 자체 안 함.
- **권고**: (a)+(b) 동시. 함정 회피 가설은 *행동 데이터*로만 검증되지 자체 분석으로 검증 안 됨. [합성]

### 모순 4: 백그라운드 자동 검출 = 차별화 ↔ 의식·동기부여 없으면 행동 변화 안 일어남

- **증거 A** [경쟁-WhiteSpace·페인#3][제품-2 옵션B 차별화]: AirPods 백그라운드 자동 검출은 "매 식사마다 앱 켜기" 14일 70% 이탈 페인을 죽인다. 제로 마찰 = 차별화 핵심.
- **증거 B** [경쟁-3.4 Eat Right Now][제품-2 옵션B MVP#3·#5]: 그러나 *측정만으로는 행동 안 바뀜*. Bite Counter가 무료인데도 안 뜨는 이유. Eat Right Now가 콘텐츠로 성공한 이유. 사용자가 *의식*해야 식사 속도가 변한다.
- **결정 필요**: "자동 검출(무의식)"과 "행동 변화(의식)"는 본질적으로 충돌한다. 어떻게 두 가지를 한 제품 안에 넣나?
- **결정 옵션**:
  - (a) **자동 측정 + *능동적 코칭 모먼트*** — 측정은 백그라운드, 그러나 *식사 중 햅틱*과 *주간 리포트 일요일*은 의식 트리거. 사용자가 *원할 때만* 의식. → 옵션 B의 MVP#3·#5 설계가 정확히 이걸 지향.
  - (b) **모드 전환** — 평소엔 자동, "오늘 의식 모드" 토글로 전환 시 능동 코치.
  - (c) **포기**: 자동 검출만 하고 행동 변화는 사용자에게 위임 — Bite Counter 함정 반복.
- **권고**: (a). 단, *햅틱 알림 수용성*이 페르소나 1·2에서 NO-GO 60% 이상이면 (b)로 fallback. 이는 인터뷰 가설 #1로 검증. [제품-6.1 질문#1·#8]

### 모순 5: 자기 투사 함정 ↔ 본 프로젝트 발의자가 핵심 페르소나

- **증거 A** [시장-거짓수요#4]: 발의자가 30대 IT 직장인 남성·혼밥+에어팟 시청 페인을 *직접* 경험 → 자기 시장을 보고 있을 가능성.
- **증거 B** [시장-페르소나#1·#2]: 그러나 1·2순위 페르소나(한지원 32세 IT 개발자, 박소연 29세 마케터)가 *발의자 인접 세그먼트와 정확히 겹침* — 즉 발의자의 페인이 *일반화 가능한 페인*일 가능성도 동시에 존재.
- **결정 필요**: 페르소나 1·2의 페인이 정말 발의자만의 페인이 아닌가?
- **결정 옵션**:
  - (a) **인터뷰 대상 다양화** — 한지원·박소연형 외 *반대 가설 그룹*(50대, 비IT, 비AirPods 사용자) 2–3명도 인터뷰에 포함.
  - (b) **결제 행동 게이트** — 설문 yes 응답 95%+이지만 결제 5%. 4주차 *실제 결제* 30%+ 게이트 [제품-6.4]만 신뢰.
  - (c) **공동창업자/외부 자문** — 발의자가 아닌 사람의 시각 입력.
- **권고**: (a)+(b). 페르소나 검증은 *결제 행동*까지 가야 진짜. [시장-거짓수요#1·#4]

| 조건 | GO 신호 | NO-GO 신호 | PIVOT 신호 |
|---|---|---|---|
| **기술 정확도** (2주 프로토타입) | 자유생활 chewing count MAPE ≤ 15% / IMU 백그라운드 60분 중단 비율 ≤ 30% | MAPE > 25% / 중단 비율 > 50% (=핵심 가치 붕괴) | MAPE 15–25% → *결과 KPI를 절대치가 아닌 상대치(어제 대비)로* 약속 변환 |

| **지불 의향** (8주 베타 결제 게이트) | N=20–30 베타에서 *4주차 실제 결제 전환율* 30%+ / 페르소나 1·2 각 5명 중 3명+ 월 4,900원 6개월 결제 의향 *행동* | 4주차 결제 < 10% / 인터뷰 결제 의향 *행동* 5명 중 1명 미만 | 결제 10–30% / 인터뷰 결제 의향 약 → Freemium BM 또는 *옵션 D 클리닉 B2B로 피벗* |

**종합 게이트**: 4행 모두 GO → **본격 GO**. 1행 NO-GO → 해당 문제 해결 후 재평가. 2행 이상 NO-GO → **NO-GO 또는 옵션 D 피벗**. [합성, 제품-6.4]

## 8. 리스크 (우선순위 순)

리스크는 *지금 결정에 영향을 주는* 순서로 정렬. 결정 임팩트 ★★★ 순.

### 리스크 1 (★★★): Apple jaw health OS 흡수 — *전략 자산이 코칭/콘텐츠가 아니라면 사망*

- **영향**: 옵션 B의 raw 측정 차별화가 OS 기능에 흡수되면 *코칭 페르소나·콘텐츠*만 남는다. 그게 자산이 아니면 시장에서 추방.
- **발생 가능성**: 중-고. 2022년·2026년 특허 두 건 누적 → Apple은 "언제"의 문제. 12–24개월 가능.
- **출처**: [기술-1.2, 5.7]
- **완화**:
  - 처음부터 *코칭 페르소나·친절 톤 카피·한국 결과 프레이밍 콘텐츠*를 핵심 자산으로 빌드 (옵션 B MVP#3·#5).
  - 18–24개월 안에 *옵션 D B2B 워크플로우* 또는 *옵션 A 의료 채널*로 진화 경로 확보 [제품-5].
  - Apple Health "친구"로 포지셔닝(Apple이 raw 데이터 주면 우리가 코치 위에 얹는다) — 적대적 관계 아니게.

### 리스크 2 (★★★): 자유생활 정확도 부족 — *백그라운드 60분 미유지 시 핵심 가치 붕괴*

- **영향**: F1 0.71–0.80은 학술 baseline. *우리 구현*이 거기 못 미치면 사용자 신뢰 붕괴. 백그라운드 IMU 스트림이 영상 60분 중 자주 끊기면 핵심 차별화(자동 검출)가 거짓말이 됨.
- **발생 가능성**: 중. iOS 정책·AirPods 펌웨어 업데이트 변동성 상존. App Store 정책 회색지대 [기술-5.6].
- **출처**: [기술-2.1, 5.1, 5.6]
- **완화**:
  - **2주 안에** 프로토타입으로 *직접 측정* (본인+지인 5명, N=20 식사). 중단 비율 30%+ 시 즉시 KPI 변환 (절대 → 상대).
  - 정확도 약속을 *임상 ≠ 코칭*으로 명확 분리. 마케팅에서 "100%"·"의료급" 어휘 절대 금지 [경쟁-피할것#1].
  - 캘리브레이션 1–2식사를 첫 사용자 경험에 명시 — 정직 = 신뢰 [기술-5.2].

### 리스크 3 (★★★): 거짓 수요 — *측정 욕구 ≠ 결제 욕구*

- **영향**: 사용자가 "건강한 식습관 관심" 95% yes 응답하지만 결제는 5%. 우리가 인터뷰 데이터를 시장 데이터로 오인하면 *MVP 빌드 후 결제 0%* 시나리오.
- **발생 가능성**: 고. 본 카테고리는 [시장-거짓수요#1·#5] 두 함정 모두 해당. "장원영처럼 천천히" 인플루언서 효과까지 시그널 오염.
- **출처**: [시장-6, 거짓수요#1·#5]
- **완화**:
  - 인터뷰 entry filter에 *"지난 3개월 헬스 앱 결제 경험"* 의무 [제품-6.1].
  - 인터뷰 질문에 *행동 결제 의향* 포함 (말이 아닌 행동 — "지금 카드로 4,900원 결제할까요?" 시뮬레이션).
  - 8주 베타에서 *실제 결제 게이트 30%+* — 행동 데이터 외 신뢰 없음 [제품-6.4].

### 리스크 4 (★★): 식이장애 트리거 위험 — *윤리·법적 리스크*

- **영향**: 측정·게이미피케이션이 식이장애 위험군에 *임상적 해*가 될 수 있음. 한 건이라도 의료적 사고 발생 시 평판·법적 손상.
- **발생 가능성**: 중. 페르소나 5(정민호) 비중 작지만 *우연 노출*은 항상 발생.
- **출처**: [시장-페르소나#5, 거짓수요#7], [경쟁-페인#5]
- **완화**:
  - 1차 마케팅 타겟에서 식이장애 회복기 *명시 제외*.
  - 식이장애 학회 자문 1건 (1–3개월 내) — 안전 가이드 적용 [제품-7].
  - 친절한 톤 (Gentler Streak) 카피 가이드 의무화. "당신은 빠르다" 어휘 금지.
  - 모드 분리 — V2 임상 모드는 의료 파트너 동행 시만.

### 리스크 5 (★★): 1인 개발자 자산 부담 — *콘텐츠·B2B·한국 모두 동시 못 함*

- **영향**: 옵션 B의 친절한 톤 카피·결과 프레이밍 콘텐츠는 1인 개발자에게 *기술 코드보다 어려운 자산*. 작가/카피라이터 부재 시 *Bite Counter 함정* (기술은 되는데 안 씀).
- **발생 가능성**: 고. 발의자가 단독 진행 시 거의 확정.
- **출처**: [경쟁-3.4 Eat Right Now Dr. Jud 개인 브랜드][제품-2 옵션B 리스크]
- **완화**:
  - 카피라이터/콘텐츠 작가 *프리랜스 1명 6개월 협업* 예산 처음부터 확보.
  - 또는 Eat Right Now 같은 *기성 임상 콘텐츠 IP 라이선스* 협상 (Dr. Jud 또는 한국 임상가).
  - V2까지는 *코어 메시지 5–10개*만 — 매주 새 콘텐츠 만들 필요 없는 minimal viable copy.

### 그 외 리스크 (한 줄)

## 9. 검증해야 할 핵심 가설 5–7개

각 가설에 *검증 방법·기간·비용*을 명시.

### 가설 1 (기술): AirPods IMU 백그라운드 60분 유지 + 자유생활 chewing count MAPE ≤ 15%
- **검증**: AirPods Pro 2세대 + tukuyo/AirPodsPro-Motion-Sampler 포크. 본인+지인 5명, N=20 식사 데이터.
- **기간**: 2주
- **비용**: 0–10만원 (지인 인센티브)
- **출처**: [기술-2.1·5.6][제품-6.2]

### 가설 2 (시장·페인): 한지원·박소연형이 지난 주 식사 속도가 *자기 건강에 부정적 영향*을 자발 발화
- **검증**: 페르소나 1·2 각 5명, 총 10명 디스커버리 인터뷰. entry filter = 지난 3개월 헬스 앱 결제 경험.
- **기간**: 1–2주
- **비용**: 30–50만원 (인터뷰 사례비 5만원/인 × 10)
- **통과 기준**: 각 페르소나에서 4명+ 자발 발화. 거짓 수요 함정 회피.
- **출처**: [시장-페르소나#1·#2, 거짓수요#1][제품-6.1 질문#5]

### 가설 3 (행동): 식사 *중* 햅틱 알림 수용성 — 부정 반응 60% 미만
- **검증**: 인터뷰 시 *영상 시청 중 햅틱 시뮬레이션* 시연 + 1–10점 평가. + 8주 베타 N=20에서 햅틱 비활성화 비율 측정.
- **기간**: 1주 (인터뷰) + 8주 (베타)
- **비용**: 인터뷰 비용에 포함, 베타 0
- **통과 기준**: 인터뷰 NO-GO 60% 미만 / 베타 햅틱 비활성화 30% 미만
- **출처**: [시장-NO-GO조건][제품-6.1 질문#1·#8]

### 가설 4 (지불): 4주차 *실제 결제* 전환율 30%+
- **검증**: 옵션 B MVP 5개 기능 빌드 → N=20–30 4주 무료 → 4주차 카드 결제 게이트.
- **기간**: 12주 (개발 6–8주 + 베타 4주)
- **비용**: 개발 자원 (1–2인 풀타임) + 베타 모집 50–100만원
- **통과 기준**: 30%+ 결제 전환 = 본격 GO / 10–30% = 옵션 D 피벗 검토 / 10% 미만 = NO-GO
- **출처**: [시장-거짓수요#1·#5][제품-6.4]

### 가설 5 (UX·차별화): 결과 프레이밍 대시보드가 씹기 카운터 대시보드보다 retention 높다
- **검증**: 베타 N=20–30 A/B (10명 결과 프레이밍 / 10명 카운터). 4주 retention·DAU 측정.
- **기간**: 8주 (개발 4주 + A/B 4주)
- **비용**: 개발 자원
- **통과 기준**: 결과 프레이밍 군 4주 retention +20pp 이상
- **출처**: [시장-거짓수요#5][제품-2 옵션B MVP#2]

### 가설 6 (의료·V2): 소화기내과 의사가 위염 환자에게 *디지털 코치 처방 의향* 있음
- **검증**: 소화기내과 의사 1–2명 자문 인터뷰 (지인망 또는 의료 커뮤니티). 한국 영양사 2–3명 추가 인터뷰.
- **기간**: 1개월
- **비용**: 자문료 30–60만원 (회당 15–30만원)
- **통과 기준**: 의사 1+ "처방 의향 있음" + 영양사 2+ "환자 데이터 원격 모니터링 가치 있음" → 옵션 A·D V2 모듈 진화 경로 확보
- **출처**: [제품-2 옵션A·D 가설][제품-6.3]

### 가설 7 (윤리): 식이장애 학회·임상가 입장 — 우리 톤·기능이 *임상 안전*인가
- **검증**: 식이장애 학회·임상가 1명 자문. 친절한 톤 카피 가이드 + 1차 타겟 제외 정책 검토.
- **기간**: 1개월
- **비용**: 자문료 20–40만원
- **통과 기준**: "안전 가이드 따르면 임상적 해 위험 낮음" 합의
- **출처**: [시장-거짓수요#7][제품-7]

## 10. 다음 단계 — 실행 가능 행동 5–7개

추상적 "더 조사" 금지. *돈·시간·사람*이 어디 가는지.

### 1주 내

1. **디스커버리 인터뷰 대상자 모집 N=10** — 지인망/링크드인/디시 슬로우이팅 후기 댓글로 페르소나 1형 5명 + 페르소나 2형 5명. entry filter = 헬스 앱 결제 경험. *인터뷰 일정 5일 내 확정*. [제품-6.1]
2. **AirPods IMU 프로토타입 1차** — tukuyo/AirPodsPro-Motion-Sampler 포크 → 본인 식사 N=5 데이터 수집 + 영상 시청 60분 백그라운드 중단 비율 1차 측정. [제품-6.2]

### 1개월 내

3. **디스커버리 인터뷰 10건 완료 + 보고서** — 8개 질문 [제품-6.1] 통과 기준으로 평가. NO-GO 시그널 발견 시 *즉시* 가설 재구축.
4. **소화기내과 의사 1–2명 + 영양사 2–3명 자문 인터뷰** — 옵션 A·D V2 진화 경로 사전 확인. 식이장애 학회 자문 1건도 동시.[제품-6.3]
5. **MVP 기능 정의서 + 카피라이터 모집** — 옵션 B 5개 기능 spec 확정 + 친절한 톤 카피 6개월 프리랜스 협업 1명 계약. [리스크 5 완화]

### 3개월 내

6. **Closed Beta 빌드 + N=20–30 모집** — 옵션 B MVP 5개 기능 (자동 검출, 결과 대시보드, 햅틱 코칭, 캘리브레이션, 주간 리포트). 베타 4주 무료 → 4주차 결제 게이트. *결과 프레이밍 vs 카운터 A/B 동시*. [제품-6.4][가설 4·5]
7. **4주차 게이트 의사결정** — 결제 전환 30%+ → 본격 GO + 옵션 A·D V2 로드맵 확정 / 10–30% → 옵션 D 피벗 검토 / 10% 미만 → NO-GO 또는 가설 재구축.

**예산 대략치 (1인 발의자 가정)**:
- 인터뷰 사례비·자문료: 100–150만원
- 카피라이터 6개월: 300–600만원
- 베타 모집: 50–100만원
- 1·2인 개발 자원: 발의자 본인 + 협업 1명 (별도 비용 산정)
- **3개월 합계**: 약 500–900만원 + 인건비
