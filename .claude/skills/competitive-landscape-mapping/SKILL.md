# Competitive Landscape Mapping

경쟁 환경을 *왜곡 없이* 매핑하는 방법론. `competitive-landscape-researcher` 에이전트가 사용한다.

## 왜 이 스킬이 필요한가

경쟁사 조사는 두 가지 함정에 빠지기 쉽다:
1. **"우리는 경쟁사가 없다"** — 거의 항상 거짓. 사용자가 *오늘* 같은 페인을 어떻게든 풀고 있다면, 그게 경쟁이다.
2. **단순 리스트** — 30개 경쟁사 이름만 나열한 보고서는 의사결정에 무용하다. *패턴*과 *시사점*이 가치다.

이 스킬은 두 함정을 모두 회피한다.

## 3계층 분류

모든 경쟁사를 3계층 중 하나에 배치한다:

### 직접 경쟁
같은 페인을 *같은 방식*으로 푸는 자.
- 예: 저작운동 자체를 트래킹하는 앱·디바이스
- 보통 적다 (3-5개 이하). 적다고 시장이 없는 게 아니라, *우리가 새 카테고리를 만들고 있다*는 신호일 수 있음

### 인접 경쟁
같은 사용자가 같은 페인을 *다른 방식*으로 풀거나, 인접 페인을 푸는 자.
- 예: 식사 일기 앱, 다이어트 코칭, 마음챙김 식사, 식이장애 케어
- 보통 많다 (15-30개). *대체 가능성*이 핵심 — 사용자가 "이미 X를 쓰니까 됐어"라고 말할 수 있는 모든 것.

### 대체재 (Substitutes)
*디지털이 아닌 방식*으로 같은 페인을 푸는 자.
- 예: 영양사 상담, 가족의 잔소리, 자가 메모, "그냥 의식하기"
- 빠뜨리기 쉽지만 가장 강력한 경쟁자일 때가 많다 (특히 "그냥 의식하기 = $0")

각 경쟁사는 *반드시* 한 계층에 배치하고, 애매하면 가장 가까운 곳 + 주석.

## 비교표 표준 컬럼

```markdown
| 이름 | 계층 | 핵심 기능 | 디바이스/센서 | BM | 가격 | 타겟 | 현재 상태 | 출처 |
|------|------|----------|--------------|----|------|------|----------|------|
```

- **현재 상태**: 활발 / 정체 / 단종 / 인수 / 피벗 — *단종된 서비스가 가장 가치 있는 데이터*다
- **출처**: 공식 사이트, 앱스토어 링크, 또는 보도자료 — 추측 금지
- 표는 8-12개 행이 적정. 30개는 너무 많고 분석을 묽게 만든다. 가장 시사점 많은 것만.

## 사용자 리뷰 마이닝 (필수)

별점 5점 리뷰는 마케팅에 가깝다. **1-2점 리뷰의 텍스트**가 페인 보이스다. 다음 출처를 반드시 확인:

- App Store 리뷰 (필터: 별점 1-2)
- Google Play 리뷰 (필터: 별점 1-2)
- Reddit 검색: `[앱이름] disappointing`, `[앱이름] alternative`
- Trustpilot, ProductHunt 댓글
- 한국: 네이버 카페 후기, 디시 갤러리, 클리앙 후기

리뷰에서 추출할 것:
- 반복되는 불만 키워드 (예: "정확도", "구독료", "동기부여")
- 사용 중단 이유
- 대안으로 옮겨간 서비스

이걸로 **사용자 페인 패턴 섹션**을 채운다.

## 케이스 딥다이브 선정

전체 비교표에서 가장 시사점 많은 3-5개를 골라 자세히 본다. 선정 기준:

1. **죽은 거인** — 한때 화제였다가 단종된 서비스 (Vessyl, Healbe GoBe2 같은 것). 실패 분석이 가장 큰 학습.
2. **현재 성공 중인 모범** — 인접 카테고리에서 현재 잘 되는 사례 (Noom 같은). UX·BM 학습.
3. **가장 가까운 직접 경쟁** — 우리가 가장 닮을 가능성이 큰 사례.
4. **흥미로운 안티-패턴** — "절대 이렇게는 하지 말자"는 사례.

각 딥다이브:
- 무엇을 시도했나
- 어떻게 작동했나 (정확한 메커니즘 / 가격 / 타겟)
- 사용자 반응 (리뷰·매체 인용)
- 성공/실패 요인 가설
- 우리에게 주는 시사점

## White Space 발견

비교표를 다 작성했으면, 다음 차원으로 *빈 공간*을 찾는다:

| 차원 예시 | 질문 |
|----------|------|
| 타겟 인구 | 청소년? 노인? 직장인 점심? 의료적 필요자? 어디가 비어있나 |
| 가격대 | $0 vs $5 vs $50? 어디가 비어있나 |
| 폼팩터 | 기존 디바이스 활용 vs 전용 디바이스? 어느 조합이 빠진 |
| 동기 메커니즘 | 데이터 시각화 vs 게이미피케이션 vs 소셜 vs 의료 처방? |
| 데이터 신뢰도 | 의료급 정확도 vs 간이 트래킹? |

차원별로 "현재 시장에서 X 영역에 있는 자 = ___" 식으로 빈 곳 노출. 이게 차별화 후보다.

## 시사점 도출 — 따라할 것 / 피할 것

리뷰 + 케이스 딥다이브 + white space에서 **5-10개의 시사점**을 뽑는다:

```markdown
**따라할 것:**
1. Noom의 점진적 행동 변화 카피 — 즉각 결과보다 습관 변경에 집중 [Noom 케이스]
2. ...

**피할 것:**
1. Vessyl처럼 정확도 부풀린 마케팅 → 출시 후 신뢰 무너짐 [Vessyl 케이스]
2. ...
```

각 시사점은 *하나의 출처*에서 도출 가능해야 한다 (인용 가능).

## 흔한 실수

- ❌ 한국 시장만, 또는 미국 시장만 보기 — 두 시장 모두 본다
- ❌ "이건 경쟁사가 아니다"라고 가볍게 제외 — 사용자가 대안으로 쓸 수 있으면 경쟁이다
- ❌ 별점 평균만 보고 "성공/실패" 판정
- ❌ 단종 사실 누락 — 한때 화제였던 서비스가 *지금* 어떻게 됐는지가 핵심 정보
- ❌ B2B 경쟁사 빠뜨리기 (영양사용 SaaS, 보험사 디지털 헬스 모듈 등)

## 후속 작업

기존 보고서가 있으면:
- 새 경쟁사 발견 시 *해당 행만 추가*
- 상태 변화(단종/인수) 시 *상태 컬럼만 업데이트*
- 새 케이스 딥다이브가 필요하면 기존 5개 중 가장 약한 것을 교체
