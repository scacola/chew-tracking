# Product Ideation

리서치를 *결정 가능한 형태*로 변환하는 방법론. `product-ideation-strategist` 에이전트가 사용한다.

## 왜 이 스킬이 필요한가

리서치는 끝났지만 "그래서 뭘 만들 건데?"는 가장 어려운 질문이다. 흔한 실패 모드:
- **컨셉 1개만** — 사용자에게 받아들이거나 거절하라는 강요
- **모호한 컨셉** — "AI 기반 마음챙김 식사 코치" 같은 표현은 의사결정 불가
- **백서가 아닌 비전 문서** — 멋있는데 실행 가능 단위가 없음

이 스킬은 **복수 옵션 + 명시적 트레이드오프 + 검증 가능 가설**로 결정을 가능하게 한다.

## 작업 진입 조건

다음 3개 파일이 모두 있어야 시작:
- `_workspace/01_tech_feasibility.md`
- `_workspace/02_competitive_landscape.md`
- `_workspace/03_market_demand.md`

없으면 즉시 멈추고 오케스트레이터에 에러 보고. 이 에이전트는 *합성*만 하지, 자체 리서치는 안 한다.

## 입력 요약 (필수 첫 단계)

작업 시작 시, 입력 3개 보고서의 핵심을 각 5줄 이내로 요약하여 출력 보고서 첫 섹션에 넣는다. 이유:
- 본인 추론의 *근거*를 추적 가능하게 만든다
- 입력 보고서의 핵심을 잘못 해석하지 않았는지 사용자가 검증 가능
- 입력이 변경되면 (후속 작업 시) 어떤 부분을 다시 합성해야 하는지 명확

요약은 단순 인용이 아니라 *이 컨셉 작업에 영향을 주는 사실*만 발췌.

## 컨셉 옵션 — 최소 3개, 진짜 다른 것

3개 옵션은 **철학적으로 다른 답**이어야 한다. 다음과 같이 *축이 다르도록*:

| 차원 | 옵션 A | 옵션 B | 옵션 C |
|------|--------|--------|--------|
| 타겟 폭 | 좁음+페인 강 (의료 필요자) | 광범위+페인 약 (일반 직장인) | B2B (전문가용) |
| 가치 제안 | 의료 보조 | 마음챙김 라이프스타일 | 환자 모니터링 SaaS |
| BM | 의료보험/처방 | 구독 | B2B 라이선스 |
| 차별화 축 | 정확도 | 행동 변화 | 워크플로우 통합 |

**나쁜 옵션 셋 (피할 것):**
- 옵션 A: 다이어트용 / 옵션 B: 다이어트용 + 게이미피케이션 / 옵션 C: 다이어트용 + 소셜
- → 모두 같은 카테고리, 같은 사용자, 같은 BM. 진짜 옵션이 아니다.

## 컨셉 1개당 필수 항목

```markdown
### 옵션 N: [작업 이름]

**한 문장 정의**: [Y(상황/페인)에 대해 X(타겟)이 Z(가치)를 얻게 한다]

**타겟 페르소나**: 시장 보고서의 페르소나 #N "한지원"형 — 정확히 인용

**핵심 페인 → 가치**: [어떤 미충족 니즈를, 어떤 메커니즘으로 푸는지]

**차별화 근거**: 경쟁사 보고서 [경쟁-WhiteSpace 섹션]의 X 영역이 비어있음 → 우리가 채움

**핵심 기능 (MVP, 3-5개)**:
1. ...
2. ...
3. ...

**부가/확장 기능 (V2+)**:
- ...

**비즈니스 모델 옵션 (1-3개)**:
- 주안: 구독 월 5,000원 (B2C)
- 대안 1: 의료기관 B2B 라이선스 ...
- 대안 2: 보험사 제휴 (검증 필요)

**검증해야 할 핵심 가설 3-5개**:
1. [페르소나]가 [핵심 페인]을 [상정한 강도]로 가지고 있다 — 인터뷰로 검증
2. AirPods IMU로 [정확도 X%] 달성 가능 — 프로토타입으로 검증
3. ...

**리스크**:
- 기술: ...
- 시장: ...
- 실행: ...
```

각 항목이 비면 그 컨셉은 미완성. 무리해서 채우지 말고 솔직히 "검증 필요" 표시.

## 차별화 — 막연함 금지

다음은 차별화가 *아니다*:
- ❌ "AI 기반"
- ❌ "더 정확한"
- ❌ "프리미엄 UX"
- ❌ "한국 시장 맞춤"

진짜 차별화는 다음 형태:
- ✅ "경쟁사 X는 카메라 기반이라 사용자가 의식적으로 켜야 함. 우리는 AirPods로 *백그라운드*에서 자동 검출 → 의식적 행동 마찰 0"
- ✅ "기존 마음챙김 식사 앱은 *식사 후* 일기를 쓰게 함. 우리는 *식사 중* 실시간 피드백 (소리/햅틱) → 행동 즉시 변경 가능"

차별화는 **경쟁사 보고서의 빈 공간 + 시장 보고서의 미충족 페인**의 *교집합*에서 도출한다.

## 비즈니스 모델 옵션 매트릭스

각 컨셉마다 BM 옵션을 1-3개 제시. 다음 매트릭스에서 어디 위치하는지 명시:

| BM 유형 | 적합 조건 | 위험 |
|---------|----------|------|
| B2C 구독 | 페인 강·일상 사용 | 이탈률 높음 (헬스 앱 평균) |
| B2C Freemium | 광범위 사용자 + 일부 프리미엄 가치 | 전환율 낮음 |
| 디바이스 번들 | 전용 하드웨어 필요시 | 초기 자본 + 유통 |
| B2B (의료) | 정확도 의료급 + 임상 근거 | 인허가 시간/비용 |
| B2B (보험) | 행동 변화 → 의료비 절감 입증 | 한국에서 검증 필요 |
| B2B (코치) | 영양사·트레이너 워크플로우 통합 | 시장 좁음 |
| 데이터 라이선스 | 익명화된 식사 데이터 가치 | 프라이버시 |

특정 BM을 권장할 땐 *왜 이 시점에 이 BM인지* 1-2문장.

## MVP 정의 — 핵심 기능만

MVP는 다음을 *동시에* 만족해야 한다:
1. 핵심 가설을 검증할 수 있다 (위 "검증해야 할 가설"의 1-2번을 직접 검증)
2. 6-12주 내 만들 수 있다
3. 한 명의 페르소나가 *돈을 낼지 안 낼지* 결정할 수 있다

MVP 기능 정의 시 *제거*가 더 중요. "있으면 좋은 것"은 V2로.

## 인접 확장 — 2년 내 현실적인 것만

확장 시나리오는 *2년 안에 갈 수 있는*, *오늘 결정에 영향을 주는* 것만:

- ✅ "B2C 성공 후 영양사용 워크플로우 SaaS로 확장 — 같은 데이터·다른 UI"
- ❌ "5년 후 글로벌 보험사 통합" — 너무 멀어서 오늘 결정 무관

각 확장 시나리오에 *진입 조건*을 명시:
- "B2C 사용자 1만 명 + 행동 변화 데이터 6개월" → 영양사 SaaS 진입 가능

## 컨셉 비교표 (필수)

옵션 모두를 5축으로 5점 만점 평가:
- 타겟 명확성
- 차별화 강도
- 기술 난이도 (역수 — 쉬울수록 점수 ↑)
- 시장 매력도
- 실행 난이도 (역수)

총점이 다가 아니라 *어느 축에서 강한지*가 의사결정 입력. 점수 옆에 1-2문장 근거.

## 추천 — 1순위와 그 이유

비교표에서 1순위를 명확히 추천. 단, *왜 다른 옵션이 아닌지*도 짧게:

```markdown
**1순위**: 옵션 B (일반 직장인 마음챙김 식사 코치)
**이유**: 시장 페르소나 #1·#2를 모두 커버 가능, 차별화는 *백그라운드 자동 검출*에 있음.
**옵션 A 대신 B인 이유**: 옵션 A(의료 보조)는 인허가·임상 근거 부족으로 1년+ 걸림.
**옵션 C 대신 B인 이유**: 옵션 C(B2B SaaS)는 사용자 워크플로우 알 수 없는 콜드 스타트.
```

## 다음 단계 — 검증 가능 행동

추상적 "더 조사해야" 금지. 다음 형태로:
- "1주 내: 타겟 페르소나 [한지원]형 사용자 5명 디스커버리 인터뷰. 질문 5-10개는 아래에 첨부"
- "2주 내: AirPods IMU로 저작 검출 프로토타입 — 본인+지인 3명 데이터 수집"
- "1개월 내: 내과/소화기 의사 자문 1-2건"

각 행동에 *어떤 가설을 검증하는지* 매핑.

## 후속 작업

- 사용자가 옵션을 선택하면 → 그 옵션 한 컨셉을 더 깊게 (페르소나 인터뷰 가이드, 기능 명세, 화면 흐름까지)
- 사용자가 새 옵션을 요청 → 기존 옵션 보존 + 옵션 D, E 추가
- 새 리서치가 들어오면 → "입력 요약" 갱신 후 영향받는 컨셉만 수정
